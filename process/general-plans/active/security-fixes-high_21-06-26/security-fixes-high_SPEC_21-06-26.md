# Specification: Security Vulnerabilities HIGH-1 to HIGH-6

## Goals and Scope
Implement secure remediations for the six high-severity security vulnerabilities identified in the Teamfair repository:
- **HIGH-1**: Deno Edge Function rate limiter fail-closed enforcement, fallback on missing configuration, and Retry-After header.
- **HIGH-2**: Python AI agent server in-memory rate limiter keying by JWT authenticated user ID (`sub` claim) instead of client-supplied student name, and extension to `/verify-task`.
- **HIGH-3**: Python agent CORS configuration tightening (restrict headers and Vercel preview regex).
- **HIGH-4**: Python agent error responses sanitization (remove raw exception details in 500 responses).
- **HIGH-5**: Python agent indirect prompt injection prevention via recursive validation of workspace data fields.
- **HIGH-6**: Deno Edge Function API responses security headers implementation.

## Technical Requirements

### HIGH-1: Deno Rate Limiter Fail-Secure
- If Upstash Redis client or rate limiter is not configured (`strictLimiter` is null), or if an error is encountered in the rate-limiting call, fail closed by denying the request.
- Return HTTP 429 status code with a standard JSON error response.
- Attach the `Retry-After` header specifying the remaining cooldown duration in seconds (dynamic for successful rate limits, default 60 seconds on unconfigured/failed check).

### HIGH-2: Python Rate Limiter JWT Keying
- Extract the authenticated user's ID via the validated JWT token (`user["sub"]` claim).
- Enforce the 30-second rate limit window keyed by this user ID.
- Apply this rate limit verification to both POST `/analyze-contribution` and POST `/verify-task`.

### HIGH-3: Python CORS Restrictions
- Restrict `allow_headers` in the CORS middleware setup to `["Content-Type", "Authorization"]`.
- Tighten `_DEFAULT_ORIGIN_REGEX` to `^https://teamfair(?:-git-[a-z0-9-]+)?-nakien0205\.vercel\.app$` to restrict wildcard Vercel previews to the official repository scope.

### HIGH-4: Exception Sanitization
- Catch blocks in endpoints `/chat`, `/analyze-contribution`, and `/verify-task` must not return raw error strings (such as `f"Agent error: {e}"`) in HTTP 500 responses.
- Log the exception details server-side using standard Python logging.
- Return generic user-friendly messages for internal server errors.

### HIGH-5: Indirect Prompt Injection Prevention
- Implement recursive validation for nested workspace data structures.
- Traverse all string fields within the `workspace` payload and run the prompt injection check on them.
- Abort and reject requests that trigger injection rules or exhibit high anomaly scores.

### HIGH-6: HTTP Security Headers
- Append HTTP security headers to all responses generated by Deno Edge Functions via `jsonOk` and `jsonError` response builders:
  - `Cache-Control: no-store`
  - `X-Content-Type-Options: nosniff`
  - `Strict-Transport-Security: max-age=31536000; includeSubDomains`
  - `X-Frame-Options: DENY`
